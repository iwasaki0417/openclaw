# 2026-03-08 (土)

## LINE Bot 障害 & 独自プラグイン開発

### 障害の経緯
- 3/7深夜〜3/8にかけてLINE Botが完全に応答不能
- ゲートウェイプロセスは起動中だったが、ログが3/7 15:59 UTCで停止（フリーズ状態）
- `openclaw gateway restart` で復旧を試みたが、LINE Webhookが200 OKを返すのにエージェントが応答しない問題が継続

### 根本原因（確定）
- **ストックLINEプラグインのバグがv2026.3.2でも未修正**
- `line-plugin-plan.md` に記載の不具合と同じ：Webhookリクエストがプラグインハンドラーに到達する前に別のハンドラーが横取りして200を返す
- 正しい署名でlocalhostに直接POSTしても、ゲートウェイログにWebhook処理が一切記録されない
- ngrokのリクエスト履歴では全て200 OKだが、エージェントは一切起動しない
- `openclaw channels status --probe` は `works` と表示するが、これはGETリクエスト（検証）のみ通るため

### 実施した対応
1. `openclaw gateway restart` → 効果なし
2. `openclaw update` (2026.3.1 → 2026.3.2) → 効果なし
3. ゲートウェイのクリーン再起動（stop→install→start）→ 効果なし
4. **独自LINEプラグインを開発・デプロイ → 成功**

### 独自プラグインの構成
- 場所: `~/.openclaw/extensions/line-custom/`（`~/.openclaw/line/` からコピー）
- プラグインID: `line-custom`
- Webhook: `/line/webhook`（ストックと同じパスを使用、LINE Developers側の変更不要）
- 設定: `channels.line` から読み取り（既存設定を流用）
- API: `api.registerHttpRoute` + `api.registerChannel` で登録
- ストックLINEプラグイン: `plugins.entries.line.enabled: false` で無効化
- `channels.line.enabled: true` のまま（独自プラグインが設定を読み取るため）

### 教訓
- `openclaw channels status --probe` の `works` は信用できない（GETのみテスト）
- ストックLINEプラグインの `monitorLineProvider` 内部でWebhookが200 OKを返すがイベントが処理されないバグがある
- 独自プラグインは `api.registerHttpRoute` で直接Webhookハンドラーを登録し、署名検証→イベント処理を自前で実装することで回避
- `openclaw update` でストックプラグインは上書きされるが、`~/.openclaw/extensions/` の独自プラグインは影響なし
- プラグイン配置は `~/.openclaw/extensions/<name>/` にディレクトリとして配置（シンボリックリンクは不可）
