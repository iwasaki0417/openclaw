# 2026-03-03 日誌

## LINE Bot 復旧（09:00-09:30）

### 問題
- 3月1日頃から LINE Bot が応答不能
- LINE プロバイダーがクラッシュループ（起動→即失敗→リトライ10回→ギブアップ）
- ngrok トンネルも heartbeat timeout で不安定

### 原因
OpenClaw `2026.2.24` → `2026.3.1` アップデート後、Gateway コードに2つのバグ:

1. **Control UI ハンドラーが POST を 405 で拒否**
   - `handleControlUiHttpRequest` がパスに関係なく全 POST を 405 で返す
   - LINE Webhook（POST）がプラグインハンドラーに到達できず

2. **プラグインルートに Gateway 認証を強制**
   - `shouldEnforceGatewayAuthForPluginPath` が全プラグインルートに Gateway トークン認証を要求
   - LINE プラットフォームからのリクエストには Gateway トークンがないため 401

### 修正（ローカルパッチ）
ファイル: `~/.nvm/versions/node/v22.22.0/lib/node_modules/openclaw/dist/gateway-cli-C7FS-lL-.js`

**パッチ1** (16916行目付近): `handleControlUiHttpRequest` で POST の場合 `return false` に変更
```
// Before: POST → 405 → return true
// After:  POST → return false (プラグインハンドラーに処理を渡す)
```

**パッチ2** (19476行目付近): `shouldEnforceGatewayAuthForPluginPath` から `isRegisteredPluginHttpRoutePath` チェックを除去
```
// Before: isProtectedPluginRoutePath(pathname) || isRegisteredPluginHttpRoutePath(registry, pathname)
// After:  isProtectedPluginRoutePath(pathname)
```

### ⚠️ 注意
- このパッチは `openclaw update` で上書きされる
- 次回アップデート時に同じ問題が再発する可能性あり
- OpenClaw に Issue/PR を出すか、アップデート後に再パッチが必要

### その他
- ngrok トンネルも再起動済み（`com.ngrok.tunnel` LaunchAgent）
- LaunchAgent plist に `LINE_CHANNEL_ACCESS_TOKEN` / `LINE_CHANNEL_SECRET` 環境変数を追加済み
